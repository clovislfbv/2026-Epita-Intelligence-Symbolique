Le problème

Un même casse-tête,
plusieurs façons de raisonner.

Le Sudoku est un problème de satisfaction de contraintes : 81 cases, des chiffres 1–9, et trois familles de règles (lignes, colonnes, blocs). Résoudre une grille générale est NP-complet, un terrain idéal pour confronter des paradigmes de l'IA symbolique.

Solveur 01 · Recherche arborescente

Backtracking MRV + forward checking.

  • Bitmask de candidats chaque case garde ses valeurs possibles sur 9 bits.
  • Heuristique MRV on branche sur la case la plus contrainte (moins de candidats).
  • Forward checking l'affectation élague immédiatement la valeur chez les 20 pairs.
  • Complet trouve toujours la solution, où un « nœud » correspond à une tentative d'affectation.
backtracking.py FULL_MASK = 0b111111111 # 1..9 def solve(grid): cell = min(empty, key=candidate_count) for v in candidates(cell): assign(cell, v) prune_peers(cell, v) # forward check if solve(grid): return True undo(cell, v) return False

Solveur 02 · Couverture exacte

Dancing Links · l'Algorithme X de Knuth.

  • Reformulation le Sudoku devient un problème de couverture exacte (729 lignes × 324 contraintes).
  • Dancing Links liste doublement chaînée où couvrir/découvrir une colonne est O(1).
  • Backtracking déterministe choisir la colonne la plus contrainte, brancher, restaurer.
  • Énumération exacte sert aussi à certifier l'unicité des grilles générées.
algorithme X # 4 familles de contraintes : # · une valeur par case # · chaque valeur 1× par ligne # · chaque valeur 1× par colonne # · chaque valeur 1× par bloc cover(col); search(k+1); uncover(col) → 324 colonnes, 729 lignes

Solveur 03 · Programmation par contraintes

CP-SAT · déclarer, pas dérouler.

  • Variables entières une variable 1–9 par case.
  • Contraintes globales trois familles d'AllDifferent : lignes, colonnes, blocs.
  • Moteur OR-Tools propagation + recherche CDCL hybride, hautement optimisé.
  • Métrique le nombre de branches explorées est remonté comme nœuds.
cp_sat.py · OR-Tools m = cp_model.CpModel() x = [[m.NewIntVar(1,9) for _ in row] for row in grid] for line in rows+cols+boxes: m.AddAllDifferent(line) solver.Solve(m) # branches → nodes
CP-SAT · ortools07
Solveur 04 · Satisfiabilité booléenne

SAT · tout réduit à des clauses.

  • Encodage 9·9·9 variable v(r,c,d) vraie ssi la case (r,c) vaut d → 729 variables.
  • Clauses « au moins une » + « au plus une » par case, ligne, colonne et bloc.
  • Encodage étendu clauses redondantes implicites qui accélèrent la propagation unitaire.
  • Solveur Glucose via PySAT, où les conflits CDCL servent de compteur de nœuds.
sat.py · PySAT / Glucose3 # v(r,c,d) : 729 variables DIMACS def var(r,c,d): return r*81+c*9+(d-1)+1 # au-moins-un / au-plus-un clauses += at_least_one(cell) clauses += at_most_one(cell) # paires Glucose3(clauses).solve()
SAT · python-sat08
Solveur 05 · Métaheuristique

Algorithme génétique · évoluer vers zéro conflit.

  • Chromosome malin chaque ligne est une permutation 1–9 respectant ses indices : les lignes sont correctes par construction.
  • Fitness compte uniquement les doublons en colonnes et blocs.
  • Évolution sélection, élitisme, croisement et mutation par échange.
  • Incomplet réussit easy/medium, se dégrade sur les grilles difficiles → taux de succès sur plusieurs graines.
genetic.py population = 200; generations = 400 elite = 10; mutation = 0.25 while gen < budget: rank_by_fitness(pop) # conflits pop = elites + crossover(parents) mutate(pop) if best.conflicts == 0: break
